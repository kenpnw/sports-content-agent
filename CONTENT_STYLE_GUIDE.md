# Content Style Guide

## Purpose

This document defines how the system should write sports content for Chinese platforms.

It is focused on:

- Hupu postgame content
- Douyin short-form recap scripts


## Shared Style Rules

### Voice

The system should sound like a sharp sports editor, not a generic assistant.

Preferred:

- direct
- energetic
- informed
- discussion-aware

Avoid:

- robotic summaries
- motivational fluff
- fake insider tone
- generic praise with no game detail


### Sentence Quality

Prefer:

- short to medium sentences
- one clear point per sentence
- concrete basketball language

Avoid:

- bloated compound sentences
- repetitive wording
- too many numbers in one line


### Factual Discipline

Every claim should be grounded in input data or explicit analysis notes.

Do not:

- invent injuries
- invent quotes
- invent lineup details
- claim a player “dominated both ends” without support


## Angle Selection Rules

Good postgame content usually centers on one main angle.

Examples of good angles:

- the true turning point
- why the winning team created better shots
- one player’s late-game takeover
- one tactical mismatch that decided the game
- a standings or playoff implication

Bad angles:

- “both teams played hard”
- “the game was exciting”
- “the winner wanted it more”


## Hupu Style Guide

### Goal

Write something that feels worth discussing in a basketball forum.

### Title Rules

Good titles:

- should state winner or main story quickly
- should sound like a postgame thread, not a news crawler
- may include score if useful

Preferred patterns:

- `赛后速递：勇士拿下比赛，湖人 110 - 118 勇士`
- `比赛复盘：湖人前三节还能咬住，勇士为什么末节突然拉开？`
- `库里末节接管比赛，勇士这场赢在什么地方？`

Avoid:

- all caps
- shock-style exaggeration
- clickbait that hides the topic


### Body Structure

Recommended order:

- opening summary
- final score and context
- key player section
- game turning point
- tactical reason
- discussion prompt

### Hupu Tone

Preferred:

- “这场最关键的不是得分爆发，而是勇士把弱侧机会打出来了。”
- “湖人前三节能跟住，末节换防质量一掉，比赛就断了。”
- “如果只看比分，可能低估了这场外线火力差距。”

Avoid:

- “这是一场非常精彩非常激烈的比赛。”
- “某某球员用行动证明了自己。”
- “球队展现了顽强拼搏精神。”


### Discussion Prompts

Good prompts:

- ask about tactical choices
- ask about player role value
- ask about whether a trend can continue

Examples:

- `库里现在是不是仍然最强的关键球持有者之一？`
- `湖人面对高投射球队时，换防是不是还不够稳定？`


## Douyin Style Guide

### Goal

Write short-form script lines that can be read aloud naturally and matched with visuals.

### Opening Hook

The first line must do one of the following:

- tell the result immediately
- point out the biggest swing
- name the decisive player

Good examples:

- `勇士赢球了，关键就发生在第四节。`
- `这场不是普通赢球，勇士把卡位战直接打明白了。`
- `库里末节一接管，比赛节奏就变了。`

Weak examples:

- `今天我们来看一场NBA比赛。`
- `这是一场非常精彩的对决。`


### Script Rhythm

Preferred structure:

- hook
- result
- two strongest observations
- end with one question or takeaway

Each scene should be easy to speak in one breath.


### Douyin Voice-Over Rules

Prefer:

- spoken language
- strong transitions
- short punchy phrases

Avoid:

- too much written language
- too much passive voice
- stacked stat dumps

Good:

- `前三节湖人还能咬住，第四节就完全不一样了。`
- `勇士把球转起来之后，湖人的协防总是慢半拍。`

Bad:

- `本场比赛呈现出较高强度的竞技对抗态势。`
- `该球员在攻防两端均有不俗表现。`


### Douyin Caption Rules

Caption should:

- restate the headline simply
- add one engagement prompt
- not repeat the full script

Good:

- `勇士拿下关键卡位战，库里末节又把比赛收了。你觉得湖人最大的问题出在哪？`


## Numeric Style Rules

Prefer:

- use only the most important numbers
- pair numbers with meaning

Good:

- `库里34分9助，末节连续命中关键球。`
- `勇士全队18记三分，这就是外线差距。`

Avoid:

- listing every box score stat in sequence
- dense strings of percentages with no explanation


## Tactical Language Rules

Prefer concrete, readable explanations:

- high pick-and-roll
- weak-side rotation
- transition defense
- second-chance points
- switch failure
- paint protection

Avoid:

- empty jargon
- fake analyst language
- unexplained advanced terms


## Sensationalism Rules

The system may be sharp and lively, but must not become dishonest.

Allowed:

- strong framing
- clear judgment
- debate prompts

Not allowed:

- fabricated conflict
- fake controversy
- misleading wording for clicks


## Reuse Rules

If the same game is written for multiple platforms:

- keep the facts identical
- change rhythm, format, and framing
- do not paste the Hupu version into Douyin or vice versa


## Quality Checklist

Before accepting generated content, check:

- is the main angle clear
- are the facts consistent with input
- does the platform voice feel natural
- is there at least one concrete basketball reason
- is the ending useful for engagement or summary
