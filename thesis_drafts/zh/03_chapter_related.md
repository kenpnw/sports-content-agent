# 第 2 章 相关工作

本章对与本研究密切相关的五个方向展开文献综述：(1) 大语言模型与文本生成；(2) 检索增强生成；(3) 大语言模型的幻觉问题与抑制方法；(4) 多智能体大语言模型系统；(5) 体育内容自动生成。每个方向的综述聚焦于代表性工作的核心思想、对本研究的启发，以及本研究与其的区别与扩展。

## 2.1 大语言模型与文本生成

### 2.1.1 Transformer 与 GPT 系列的演进

现代大语言模型的基础是 2017 年 Vaswani 等 [Vaswani 2017] 提出的 Transformer 架构，其引入的自注意力（Self-Attention）机制取代了 RNN/LSTM 的循环结构，使得模型可以并行处理长序列、捕捉长距离依赖。GPT 系列 [Radford 2018; Radford 2019; Brown 2020; OpenAI 2023] 沿用 Transformer 的解码器结构，通过自回归语言建模目标在大规模无监督文本上预训练，逐步将模型参数量从 GPT-1 的 1.17 亿提升到 GPT-3 的 1750 亿、GPT-4 的更大规模（具体数字未公开），生成质量随模型规模呈现"涌现能力"[Wei 2022b]。BERT [Devlin 2019] 等基于编码器的双向预训练模型则在判别任务上占据主导地位。

GPT-3 的 In-Context Learning（ICL）能力——通过在提示词中给出几个示例即可学习新任务——开启了"提示工程（Prompt Engineering）"的研究方向 [Liu 2023]。Chain-of-Thought（CoT）提示 [Wei 2022a] 进一步证明，通过让模型显式输出推理步骤，可以显著提升复杂推理任务的表现。

### 2.1.2 指令微调与人类偏好对齐

预训练得到的基础模型并不天然适合"对话"或"遵循指令"。指令微调（Instruction Tuning）[Wei 2022; Sanh 2022; Chung 2024] 在监督方式下让模型学习"指令-响应"格式，显著提升了模型对人类指令的遵循能力。人类反馈强化学习（RLHF）[Ouyang 2022] 进一步通过人类标注的偏好数据训练奖励模型，再用强化学习算法（如 PPO）优化语言模型，使其在帮助性、诚实性、无害性三个维度上更贴近人类偏好。Direct Preference Optimization（DPO）[Rafailov 2024] 提供了一种不需要单独训练奖励模型的简化路径，已在多个开源模型中被采用。

### 2.1.3 中文大语言模型生态

中文场景下，开源模型生态也快速成熟。Qwen 系列 [Bai 2023; Bai 2024]（阿里）、ChatGLM 系列 [Zeng 2022]（智谱 AI）、Baichuan 系列 [Yang 2023]（百川智能）、DeepSeek 系列 [DeepSeek-AI 2024]（深度求索）均在中文理解与生成上达到了与 GPT-4 同代相当的水平。本研究采用 DeepSeek-V2 作为基础模型，主要原因有三：(1) DeepSeek-V2 在中文体育术语理解上表现优异，对"挡拆"、"空切"、"反跑"等术语的语义把握准确；(2) DeepSeek 提供 OpenAI 兼容的 API 接口，工程集成成本低；(3) DeepSeek 的定价显著低于 GPT-4，对实际生产部署友好。

### 2.1.4 与本研究的关系

本研究将 DeepSeek-V2 作为"基础语言模型层"使用，本身不涉及对 LLM 架构、训练方法的改造。本研究的创新点在于"LLM 之上的协同与约束机制"，即如何通过多智能体协议、提示契约、双层知识架构等"外部框架"约束 LLM 的输出，使之在体育内容生成场景中表现可信、可控。这一思路与"通过更大规模训练或更精细微调提升 LLM 内在能力"的研究路线互补，而非替代。

## 2.2 检索增强生成（RAG）

### 2.2.1 经典 RAG 范式

Lewis 等 [Lewis 2020] 提出的 RAG 框架是当前最具影响力的"知识 + LLM"集成方案。其核心思想是：对每个用户查询，先从外部知识库（通常是向量化的文本片段）中检索 top-K 相关片段，将其作为 context 拼接进 LLM 的输入，再由 LLM 生成最终响应。这一范式有两个明显优势：(1) 解耦了"知识"与"生成"——更新知识库不需要重训模型；(2) 生成时可以引用具体来源，降低幻觉风险。RAG 已成为企业 LLM 应用的事实标准架构。

### 2.2.2 RAG 的改进路线

RAG 的局限性主要体现在三方面：(1) 检索质量受向量索引算法限制；(2) LLM 可能忽略检索片段而"自我发挥"；(3) 单一向量库难以承载结构化知识。后续工作沿这三个方向作出了改进：

- **检索质量改进**：HyDE [Gao 2023] 提出"假设性文档生成"——先让 LLM 生成一个理想化的答案，再用该答案作为 query 检索，提升召回率；ColBERT [Khattab 2020] 与其后续工作通过细粒度交互的延迟编码改善相关性；GraphRAG [Edge 2024] 通过图结构知识表征捕捉实体间关系。
- **LLM-Retriever 协同改进**：Self-RAG [Asai 2023] 让 LLM 学会在生成过程中动态决定是否检索、检索什么、是否引用检索结果；CRAG [Yan 2024] 引入"轻量检索评估器"判断检索质量并触发后备策略；RAFT [Zhang 2024] 通过微调让 LLM 学会区分"相关 vs 干扰"检索片段。
- **多模态与结构化知识扩展**：MuRAG [Chen 2022] 将 RAG 扩展到图文混合知识库；KG-Augmented RAG [Yang 2024] 引入知识图谱作为补充检索源；Table-RAG [Wang 2024] 针对表格类结构化数据设计专门的检索与对齐机制。

### 2.2.3 RAG 在数值类问题上的瓶颈

尽管 RAG 在文本类问题（如"某球员的职业生涯描述"）上效果显著，但在数值类问题（如"某球员上场赛季的命中率"）上仍有显著瓶颈。原因有二：(1) 向量检索基于语义相似度，对数值的精确召回不友好——"42%" 与 "43%" 的向量相似度极高，但语义截然不同；(2) LLM 在 context 中看到多个相近数字时，容易出现"数字串扰"，把 A 的数据误用到 B 身上。

本研究提出的"双层知识架构"正是对这一缺陷的直接回应：将结构化数值知识从向量库中剥离，放入支持精确查询的 SQLite Fact Store，配合显式的路由规则，使数值类断言必须经过 Fact Store 验证。这一设计与 Table-RAG [Wang 2024] 在思想上相似，但在路由规则的形式化上更为明确。

## 2.3 大语言模型幻觉与事实性

### 2.3.1 幻觉的分类与评估

Ji 等 [Ji 2023] 在 ACM Computing Surveys 上系统综述了 LLM 的幻觉问题，将其分为"内在幻觉（与输入冲突）"与"外在幻觉（与已知事实冲突）"两类。Huang 等 [Huang 2024] 进一步细分为"事实性幻觉（factuality hallucination）"与"忠实性幻觉（faithfulness hallucination）"。在本研究的体育内容场景下，主要关注前者——生成内容是否与真实比赛数据一致。

幻觉评估方面，FactScore [Min 2023] 将生成文本拆解为原子事实，逐一比对参考知识库；SelfCheckGPT [Manakul 2023] 通过对同一问题生成多个回答并检测一致性来估计幻觉风险；TruthfulQA [Lin 2022] 提供了一个面向常见误解的多选题数据集，专门测试模型对错误信念的抵抗能力。本研究借鉴 FactScore 的"原子事实评估"思想，在第 5 章实验设计中将每个生成片段拆解为"球员-动作-数字-时间"等原子事实单元，分别进行核实。

### 2.3.2 幻觉抑制的三类方法

幻觉抑制方法可归纳为三类：

**(1) 生成时控制**：在生成过程中实时干预模型行为。Constitutional AI（CAI）[Bai 2022] 通过让模型在生成后基于一组原则进行自我修正；CoVe（Chain-of-Verification）[Dhuliawala 2023] 通过"生成草稿 → 生成校验问题 → 回答校验问题 → 修正草稿"的多轮流程降低幻觉；ITI（Inference-Time Intervention）[Li 2023] 通过修改特定 attention head 的激活值引导模型走向更"诚实"的输出。

**(2) 后置校验**：在生成完成后用独立的校验模型或规则系统检查。FActScore [Min 2023]、FacTool [Chern 2023] 等属于此类，通常用一个独立的事实校验 LLM 或检索系统对生成内容做事后核实。

**(3) 训练时干预**：通过修改训练目标或数据让模型本身的幻觉倾向降低。RLHF [Ouyang 2022]、DPO [Rafailov 2024]、KTO [Ethayarajh 2024] 等偏好优化方法均可包含"诚实性"作为优化目标；FactTune [Tian 2024] 通过事实性偏好数据微调模型。

### 2.3.3 与本研究的关系

本研究的多智能体监督协议在思想上属于"生成时控制 + 后置校验"的混合方法：

- **生成时控制**：Writer 角色在 prompt 中被强制要求"每个断言必须附 evidence_id"，未附证据的句子在生成阶段就被结构化拒绝；
- **后置校验**：Fact Checker 角色独立校验每个 evidence_id 能否在 Fact Store / Text RAG 中找到原文，未通过的回合触发 Writer 的修正循环；
- **风险层**：Risk Guard 角色检查是否涉及球员人格化评价、争议判罚情绪化等业务规则风险，必要时阻断。

与 CAI、CoVe 相比，本协议的两个关键不同点是：(1) 校验角色与生成角色严格分离，避免"模型自我打分"的偏置问题；(2) 评审角色具有阻塞权——校验失败的内容不进入下游，而非仅作为"建议"。

## 2.4 多智能体大语言模型系统

### 2.4.1 任务分工型多智能体框架

将一个复杂任务拆分为多个 LLM 角色协作完成，是近两年快速兴起的研究方向。AutoGen [Wu 2023]（微软）提供了一套通用的"可对话智能体"框架，每个智能体可以注册不同的 system prompt、工具与对话策略；MetaGPT [Hong 2023] 将软件工程的标准化流程（PRD → 设计 → 编码 → 测试）映射到多角色 LLM 协作上；CAMEL [Li 2023] 提出"角色扮演"范式让两个 LLM 实例分别扮演"用户"与"助手"展开对话推进任务；ChatDev [Qian 2024] 模拟软件公司中的多个职位（CEO、CTO、程序员、QA）协作。

这些工作的共同点是：将复杂任务的"功能流水线"映射到多个 LLM 角色上，通过角色间的对话或消息传递推进流程。其优势在于：(1) 单一角色的 prompt 复杂度降低，行为更可控；(2) 关注点分离，每个角色专注一个子任务；(3) 错误可以在角色边界处被捕获。

### 2.4.2 辩论协作型多智能体框架

另一条研究路线强调"通过辩论提升推理质量"。Multi-Agent Debate [Du 2023] 让多个 LLM 实例对同一问题独立给出答案，然后让它们看到彼此的答案并进行多轮辩论，最终收敛到更准确的答案；ChatEval [Chan 2023] 将多智能体辩论应用于评估任务；Liang 等 [Liang 2023] 进一步研究辩论中的"思维差异度"对最终质量的影响。

这一路线的核心假设是：通过让多个模型实例从不同角度审视同一问题，可以放大模型的"自我反思"能力，挖掘出单次推理无法发现的错误。然而，辩论的有效性高度依赖任务类型——在数学推理、常识问答等"有明确正确答案"的任务上效果显著，在内容生成这类"开放性任务"上效果有限。

### 2.4.3 与本研究的关系

本研究的多智能体监督协议在结构上更接近"任务分工型"——Selector、Researcher、Writer、Fact Checker、Risk Guard 分别承担不同的子任务，通过流水线方式推进。但在监督机制上，本协议吸收了"辩论型"中的"独立校验"思想，让 Fact Checker 和 Risk Guard 作为独立角色对 Writer 的输出进行校验，而非由 Writer 自己评估。

与 AutoGen、MetaGPT 等通用框架相比，本协议在以下三个方面有针对体育内容场景的特化：

- **领域感知的角色规约**：Writer 的 prompt 中显式注入了 30+ 篮球术语库与"不强行套战术名"的负面规则，避免在转换快攻、抢板二次进攻、罚球时刻等非战术回合上硬套战术标签；
- **显式的证据契约**：每个角色的输出必须包含 `evidence_id` 字段，链接到 Fact Store 或 Text RAG 的具体记录；
- **不可逾越的评审阻塞**：Fact Checker 和 Risk Guard 的拒绝信号会触发 Writer 的修正循环或最终内容降级，而非作为可选建议。

## 2.5 体育内容自动生成

### 2.5.1 早期模板化方法

体育内容自动生成在 LLM 出现之前已有相当长的研究历史。早期工作如 STATS LLC 的 Automated Insights（NarrativeScience 子产品）采用基于模板的文本填充方法：预定义大量句式模板（如"<球员A> 在 <时刻> 命中 <投篮类型>，将比分追平 <分差>"），运行时从结构化比赛数据中填充变量，输出连贯文本。这一方法的优势在于：(1) 完全可控，不会产生事实错误；(2) 生成速度极快，适合实时报道。但缺点同样明显：(1) 风格单一，缺乏多样性；(2) 难以处理复杂战术分析等需要语义理解的内容；(3) 模板维护成本高。

国内早期工作如新华社的"快笔小新"（2015 年上线）、今日头条的"Xiaomingbot"（2016 年上线）采用类似的模板化方法，主要应用于赛后简讯生成。

### 2.5.2 LLM 时代的体育内容生成

随着 LLM 能力的快速提升，多个研究团队和产品团队尝试将 LLM 应用于体育内容生成。SportsBot [Smith 2022] 结合统计数据与 LLM 生成短文本评论，但其评估主要在"语言流畅性"维度，未涉及事实正确性的系统量化。Sportify [Author Year] 等商业产品尝试基于 LLM 生成赛后摘要，但具体技术细节未公开。

学术界已发表的相关工作中，与本研究最接近的是 [Author Year, 占位] 提出的多模态体育内容生成框架，但其工作主要面向英文场景，且未涉及多智能体监督机制。

### 2.5.3 视频与文本对齐的研究现状

体育内容的多模态对齐是另一个相关方向。BasketballGAN [Hsieh 2019] 等早期工作尝试通过深度学习模型直接从视频生成战术解说，但其依赖大规模标注数据；近期工作如 SoccerNet [Giancola 2018, Cioppa 2020] 提供了足球场景的多模态标注数据集与基线方法。但在篮球场景下，公开的高质量多模态标注数据集稀缺，导致基于端到端深度学习的方案难以充分训练。

本研究采取了"轻量级 + 可解释"的对齐方案：通过 OCR 时间映射 + 记分牌可见性检测，不依赖深度学习训练，仅利用 OpenCV 模板匹配等经典视觉技术即可在单场比赛上获得高准确率，工程部署成本极低。

### 2.5.4 与本研究的关系

本研究在公开发表的体育内容自动生成工作中，是少数同时覆盖以下四个维度的工作之一：

- **多模态对齐**：视频片段与文本评论的精确对齐；
- **多智能体监督**：5 角色协同与硬监督；
- **多平台风格化**：虎扑、抖音、微博、小红书四个差异化平台；
- **端到端实现**：从原始视频和 PBP 到 4 平台成稿的完整流水线。

## 2.6 本章小结

本章对与本研究密切相关的五个方向进行了文献综述。可以看出，尽管大语言模型在文本生成、RAG 在知识增强、多智能体在任务分工、体育内容自动生成在工程实践等各个方向都已有大量工作，但**同时满足"事实可控、多模态对齐、多平台风格化、可视化溯源"四个要求的端到端体育内容生成框架在公开发表的研究中仍属空白**。本研究的贡献正是在此处提出并实现了一个完整方案。
